# Widget de Suporte — Documentação do Usuário

## O que é?

O **Widget de Suporte** é um botão flutuante presente em todas as telas do sistema [Toriq](https://toriq.com.br). Ele permite que qualquer usuário logado abra um **ticket de suporte** diretamente de onde estiver, sem precisar sair da tela atual.

O widget aparece como um botão arredondado com o ícone de fone de ouvido e o texto **"Suporte"**, posicionado no canto inferior direito da tela.

---

## Para que serve?

O Widget de Suporte permite:

- Reportar **bugs e erros** encontrados no sistema
- Enviar **dúvidas** sobre funcionalidades
- Fazer **sugestões** de melhorias
- Relatar **problemas técnicos** (lentidão, tela travada, etc.)
- Abrir chamados sobre questões **financeiras**
- Enviar qualquer **outro** tipo de solicitação

Cada ticket enviado é automaticamente direcionado para a equipe responsável com base no seu perfil de acesso.

---

## Quem pode usar?

Todos os usuários logados no sistema podem usar o Widget de Suporte:

| Perfil | Ticket é enviado para |
|--------|----------------------|
| **Empresa SST** | Administração Vertical ON (Toriq) |
| **Instrutor** | Empresa SST ou Parceira vinculada |
| **Cliente Final** | Empresa SST responsável |
| **Empresa Parceira** | Empresa SST responsável |
| **Admin Vertical** | Administração interna |

---

## Como usar o Botão Flutuante

O botão flutuante possui 3 interações:

| Ação | O que acontece |
|------|---------------|
| **Clique simples** | Abre o formulário de suporte |
| **Arrastar** | Move o botão para outra posição na tela |
| **Duplo clique** | Reseta o botão para a posição original (canto inferior direito) |

### Mover o botão
1. Clique e segure o botão **"Suporte"**
2. Arraste para qualquer posição na tela
3. Solte o botão na posição desejada
4. O botão permanecerá na nova posição

### Resetar posição
1. Dê um **duplo clique** rápido no botão
2. O botão voltará automaticamente para o canto inferior direito

## Passo a Passo: Abrir um Ticket de Suporte

### 1. Abrir o Widget

1. Clique no botão flutuante **"Suporte"** (ícone de fone de ouvido)
2. O formulário de suporte será aberto como uma janela flutuante

### 2. Verificar Informações Automáticas

O widget preenche automaticamente:
- **Solicitante** — Seu nome e e-mail
- **Localização** — Módulo e tela onde você está no momento
- **Perfil** — Seu tipo de acesso (badge colorido)

> Essas informações ajudam a equipe de suporte a entender o contexto do seu problema.

### 3. Classificar o Problema

Preencha os campos de classificação:

#### Tipo do Problema (obrigatório)
Selecione o tipo que melhor descreve sua solicitação:

| Tipo | Quando usar |
|------|------------|
| **Bug / Erro** | Algo não funciona como deveria |
| **Dúvida** | Precisa de ajuda para usar uma funcionalidade |
| **Sugestão** | Tem uma ideia de melhoria para o sistema |
| **Problema Técnico** | Sistema lento, tela travada, erro de carregamento |
| **Financeiro** | Questões sobre cobrança, plano, pagamento |
| **Outro** | Qualquer outro assunto |

#### Prioridade
Indique a urgência do problema:

| Prioridade | Significado | Indicador |
|-----------|------------|-----------|
| **Baixa** | Pode esperar, não é urgente | Cinza |
| **Média** | Importante, mas não urgente (padrão) | Azul |
| **Alta** | Precisa de atenção rápida | Laranja |
| **Crítica** | Urgente, precisa de resolução imediata | Vermelho |

#### Impacto Operacional
Indique como o problema afeta sua operação:

| Impacto | Descrição |
|---------|-----------|
| **Nenhum impacto** | Não afeta a operação |
| **Baixo impacto** | Tenho um workaround (consigo contornar) |
| **Médio impacto** | Afeta parcialmente minha operação |
| **Alto impacto** | Afeta muito a operação |
| **Crítico** | Empresa parada! |

### 4. Descrever o Problema

Preencha os campos de detalhes:

#### Título (obrigatório)
Um resumo curto e claro do problema.
- **Bom exemplo**: "Botão de salvar não funciona na tela de cadastro"
- **Mau exemplo**: "Não funciona"

#### Descrição (obrigatório)
Descreva o problema com o máximo de detalhes possível:
- O que você estava fazendo?
- O que aconteceu?
- O que deveria ter acontecido?
- O problema acontece sempre ou só às vezes?

### 5. Adicionar Evidências (opcional, mas recomendado)

O widget oferece 4 formas de adicionar imagens como evidência:

#### Opção A: Selecionar Elemento na Tela
1. Clique no botão **"Selecionar Elemento"** (ícone de mira)
2. O widget será minimizado e uma barra laranja aparecerá no topo: "🎯 Clique no elemento com problema"
3. Passe o mouse sobre os elementos da tela — eles serão destacados com borda laranja
4. Clique no elemento que apresenta o problema
5. O sistema captura automaticamente uma **screenshot do elemento** e adiciona informações técnicas na descrição
6. Para cancelar, pressione **Esc** ou clique em **"Cancelar"**

#### Opção B: Anexar Imagem do Computador
1. Clique no botão **"Imagem"** (ícone de imagem com +)
2. Selecione uma imagem do seu computador
3. A imagem aparecerá como miniatura na seção de evidências

#### Opção C: Capturar Screenshot
1. Clique no botão **"Screenshot"** (ícone de câmera)
2. O navegador pedirá permissão para compartilhar a tela
3. Selecione a janela ou aba desejada
4. O screenshot será capturado e adicionado automaticamente

#### Opção D: Colar Imagem (Ctrl+V)
1. Copie uma imagem para a área de transferência (ex: Print Screen, recorte)
2. Com o widget aberto, pressione **Ctrl+V**
3. A imagem será colada automaticamente como evidência

> **Limite**: Máximo de **3 imagens** por ticket. O contador "X/3 imagens" mostra quantas já foram adicionadas.

### 6. Gerenciar Imagens Anexadas

As imagens anexadas aparecem como miniaturas. Ao passar o mouse sobre uma miniatura:
- Clique no ícone de **lupa** para visualizar a imagem em tamanho real
- Clique no ícone de **lixeira** para remover a imagem

### 7. Enviar o Ticket

1. Confira todas as informações preenchidas
2. Clique no botão **"Enviar Ticket"**
3. Aguarde o envio (o botão mostrará "Enviando...")
4. Uma tela de confirmação aparecerá: **"Ticket Enviado!"**
5. O widget será minimizado automaticamente após 2 segundos

> O botão "Enviar Ticket" só fica habilitado quando os 3 campos obrigatórios estão preenchidos: **Tipo**, **Título** e **Descrição**.

## Controles da Janela do Widget

A janela do formulário de suporte possui controles no cabeçalho:

| Controle | Ação |
|----------|------|
| **Arrastar pelo cabeçalho** | Move a janela para outra posição |
| **Botão minimizar (—)** | Minimiza a janela (mostra apenas o cabeçalho) |
| **Botão fechar (X)** | Fecha o widget e limpa o formulário |
| **Cantos inferiores** | Arraste para redimensionar a janela |

### Minimizar e Expandir
- Clique no botão **"—"** para minimizar
- Quando minimizado, aparece a mensagem "Clique em — para expandir"
- Clique novamente no **"—"** para expandir

### Redimensionar
- Posicione o mouse nos **cantos inferiores** da janela (esquerdo ou direito)
- O cursor mudará para indicar redimensionamento
- Arraste para ajustar o tamanho
- **Tamanho mínimo**: 320 × 400 pixels
- **Tamanho máximo**: 600 × 800 pixels

---

## Após o Envio

Depois de enviar o ticket:
- Você receberá uma confirmação visual na tela
- O ticket será registrado no sistema com todas as informações
- A equipe de suporte receberá a notificação
- Você pode acompanhar o status do ticket na **tela de tickets** do sistema

---

## Dicas

- **Seja específico no título** — Isso ajuda a equipe a priorizar e resolver mais rápido
- **Sempre adicione evidências** — Uma imagem vale mais que mil palavras
- **Use o Seletor de Elemento** — É a forma mais precisa de mostrar onde está o problema
- **Informe o impacto real** — Isso ajuda a equipe a priorizar corretamente
- **Descreva os passos para reproduzir** — "Cliquei em X, depois em Y, e apareceu o erro Z"
- **Use Ctrl+V** — Se você já tirou um print screen, basta colar direto no widget
- O widget detecta automaticamente em qual tela e módulo você está — não precisa informar manualmente
- Você pode mover o botão de suporte se ele estiver atrapalhando sua visualização
